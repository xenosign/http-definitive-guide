## ch 11 ~ ch12 퀴즈

### 문제. 제공된 '12. 기본 인증' 텍스트의 내용을 바탕으로 <보기> 중 옳지 않은 것을 고르시오.

1. 서버는 사용자 식별이 필요한 경우, 401 Unauthorized 상태 코드와 함께 WWW-Authenticate 헤더에 필요한 인증 정보를 담아 클라이언트에게 보낸다.
2. 보안 영역(realm)은 서버 내의 리소스들에 대해 서로 다른 접근 권한이나 조건을 적용해야 할 때, 이를 그룹화하여 관리하는 데 사용된다.
3. Base-64 인코딩은 바이너리 데이터 등을 시스템에서 안전하게 처리하기 위해 알파벳으로 변환하는 방식이며, 3byte의 데이터를 4개의 문자로 변환하기 때문에 전송 용량이 약 33% 증가한다.
4. HTTP 기본 인증은 사용자 이름과 비밀번호를 Base-64로 인코딩하여 전송하는데, 이는 일종의 암호화 기술이므로 제3자가 헤더를 탈취하더라도 원래의 비밀번호를 알아낼 수 없다.
5. HTTP 기본 인증은 메시지 자체를 보호하지 못하므로, 보안 결함을 해결하기 위해 통신 채널 자체를 암호화하는 SSL(HTTPS)과 함께 사용하는 것이 보편적이다.

## ch13 ~ 14 퀴즈

### 문제: 다음 중 제공된 텍스트의 내용과 일치하지 않는(옳지 않은) 것은 무엇입니까? (4)

1. 다이제스트 인증은 비밀번호를 평문으로 전송하지 않기 위해 MD5와 같은 단방향 해시 함수를 이용해 요약(digest)을 생성하며, 재전송 공격을 방지하기 위해 서버로부터 일회용 증표인 '난스(nonce)'를 받아 사용한다.

2. 공개키 암호법(비대칭키)은 인코딩과 디코딩에 서로 다른 키를 사용하며, N개의 노드가 서로 은밀하게 대화하기 위해 필요한 키의 개수가 $O(N^2)$인 대칭키 방식과 달리 $O(N)$의 키만으로도 전체 네트워크의 보안 통신이 가능하다.

3. RSA 알고리즘은 큰 소수의 곱을 구하기는 쉽지만 이를 다시 소인수분해하는 것은 매우 어렵다는 수학적 원리를 이용하며, 전자 서명에 사용할 때는 서버의 개인키로 암호화하고 클라이언트가 공개키로 복호화하여 신원을 검증한다.

4. HTTPS는 HTTP 메시지를 전송하기 전에 SSL/TLS 계층에서 암호화하는데, 이때 효율성을 위해 핸드셰이크 과정부터 실제 데이터를 전송하는 전 과정에서 오직 공개키(비대칭키) 암호화 방식만을 사용하여 통신한다.

5. 클라이언트가 HTTPS 통신을 위해 프락시를 거칠 때, 프락시는 암호화된 요청 헤더를 읽어 목적지를 파악할 수 없으므로, 클라이언트는 CONNECT 메서드를 사용하여 평문으로 목적지 호스트와 포트를 프락시에 미리 알려주어야 한다.

## ch20 퀴즈

### 문제 : 다음 중 HTTP 리다이렉션 및 부하 균형 기술에 대한 설명으로 옳지 않은 것은?

1. HTTP 리다이렉션은 서버 부하와 거리를 계산해 최적의 서버로 안내할 수 있지만, 페이지 접근 시 결과적으로 2번의 왕복(Round-trip)이 필요하여 지연이 발생한다는 단점이 있다.

2. DNS 라운드 로빈 방식은 하나의 도메인에 여러 아이피를 결부시켜 부하를 분산하지만, DNS 캐싱 효과로 인해 부하 균형이 완벽하게 이루어지지 않을 수 있다.

3. IP 주소 포워딩(NAT) 중 '완전 NAT' 방식은 패킷의 출발지 IP를 스위치의 주소로 변경하여 통신하므로, 서버가 클라이언트의 실제 주소를 알 수 없어 인증이나 결제 시 문제가 발생할 수 있다.

4. WPAD(웹 프락시 자동발견 프로토콜)는 DHCP나 DNS를 통해 PAC 파일의 위치를 찾아 프락시 설정을 동적으로 수행하며, 보안상의 이유로 주로 사설망 내부에서 사용된다.

5. ICP(인터넷 캐시 프로토콜)는 HTTP/1.0 이상의 환경에서 URL뿐만 아니라 모든 요청 및 응답 헤더를 사용하여 주변 캐시에 콘텐츠 존재 여부를 상세히 질의하기 위해 설계되었다.
